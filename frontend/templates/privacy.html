Privacy & Data

Full transparency about what RepoGator stores and how you can control it.

What We Store

Data Type What It Is Retention
GitHub profile Name, email, avatar (from OAuth) Until account deletion
Webhook events GitHub event payloads processed by agents {{ retention_days }} days after processing
Knowledge base Files and URLs you upload Until you delete them
Audit log System actions and agent outputs {{ retention_days }} days
API keys Your OpenRouter / OpenAI keys Stored encrypted, until account deletion

Where Your Data Goes

  • › Your API keys: When you provide your own OpenRouter / OpenAI keys, repository content sent to LLMs is billed directly to your accounts. RepoGator never sees your billing or usage details.
  • › Admin fallback: If you have not configured personal API keys and are an admin, content is processed via the RepoGator admin account. Non-admin users without keys will receive an error and cannot use agents.
  • › Knowledge base: Documents you upload are stored in ChromaDB on the RepoGator VPS in a collection isolated to your account — no other user can query your documents.
  • › No third-party sharing: Your data is never sold, rented, or shared with third parties. The only external calls are to GitHub (OAuth + webhooks) and the AI providers you configure.

Security Practices

  • ✓ Per-repository unique HMAC webhook secrets — GitHub events are cryptographically verified before processing
  • ✓ Per-user ChromaDB collection isolation — no query can read another user's knowledge base
  • ✓ GitHub OAuth with minimal incremental scopes — we only request permissions when features require them
  • ✓ Session cookies signed with itsdangerous — tampered or expired sessions are rejected
  • ✓ Append-only audit log for all significant system and agent actions
  • ✓ API keys stored encrypted at rest in the database
